ElasticSearch-Full Text Queries

Match Query

match查询接收text/numberics/dates,分析它们并且组成一个查询字句。

match

match查询是boolean类型的。这意味着分析提供的文本，在分析过程中根据提供的文本构造布尔查询。operator字段可以被设置成or或者and来控制布尔字句(默认是or)。

match_phrase

match_phrase和match类似但是被用来精确匹配短语。

match_phrase_prefix

match_phrase_prefix和match_phrase类似只是它允许文本中最后一个术语的前缀匹配，比如查询短语”this is a”，那么他首先会使用match_phrase来匹配this is开头的文档，然后在这些匹配的文档中找出所有以a为前缀的文档：

multi match query

multi_match允许我们在输入一个搜索内容的时候，支持在某个索引类型下的多个字段中进行搜索。

multi_match会根据fields的顺序从中搜索匹配的文档，例如我们在使用Google的时候，想要查找ElasticSearch相关的内容，那么我们希望如下规则：

  1. 如果标题中包含ElasticSearch则优先取该答案
  2. 如果标题中不包含而内容包含ElasticSearch则取该答案
